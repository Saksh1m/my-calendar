1. INTRODUCTION
================================================================================

The Unified Deadline Management System, branded as "DeadlineHub", is a
web-based application designed to help students and interns manage deadlines
scattered across multiple platforms such as college portals, Microsoft Teams,
and personal reminders. The application provides a single, centralized
dashboard where users can track all academic and internship-related tasks
and deadlines in one place.

This report documents the frontend implementation of the application, built
using React.js with Bootstrap for responsive UI design.

================================================================================
2. PROBLEM STATEMENT
================================================================================

Students and interns often juggle deadlines from multiple sources:

  - College portals (assignments, quizzes, exams)
  - Internship tools (Microsoft Teams meetings, planner tasks)
  - Personal goals (reading, fitness, portfolio updates)

Managing these across different platforms leads to:

  - Missed deadlines due to lack of a unified view
  - Difficulty in prioritizing tasks across contexts
  - No visibility into productivity patterns
  - Constant context-switching between platforms

There is a need for a centralized system that aggregates all deadlines,
provides smart reminders, and offers productivity insights.

================================================================================
3. OBJECTIVES
================================================================================

  1. Provide a unified calendar dashboard displaying tasks, quizzes,
     meetings, and submission deadlines in monthly, weekly, and daily views.

  2. Enable users to add, edit, delete, and mark tasks as completed with
     details like title, description, source, deadline, priority, and links.

  3. Implement visual urgency categorization (Critical, Approaching, Safe,
     Overdue) based on deadline proximity.

  4. Integrate with Microsoft Teams via Microsoft Graph API to sync
     meetings, planner tasks, and calendar events.

  5. Provide productivity analytics showing completed tasks, missed
     deadlines, and weekly/monthly progress.

  6. Implement a reminder system with browser notifications and email
     alert configuration.

  7. Build a responsive interface that works on desktop and mobile devices.

5. SYSTEM ARCHITECTURE
================================================================================

The frontend follows a component-based architecture with the following layers:

  +-------------------------------------------------------------------+
  |                        Browser (Client)                           |
  +-------------------------------------------------------------------+
  |  React Router  -->  Layout (Navbar + Sidebar + Content Area)      |
  +-------------------------------------------------------------------+
  |  Pages/Routes:                                                    |
  |    /            -->  Calendar Dashboard                           |
  |    /tasks       -->  Task Manager                                 |
  |    /analytics   -->  Productivity Analytics                       |
  |    /teams       -->  Microsoft Teams Integration                  |
  |    /reminders   -->  Reminder Settings                            |
  +-------------------------------------------------------------------+
  |  State Layer:  TaskContext (Context API + useReducer)              |
  +-------------------------------------------------------------------+
  |  Data Layer:   Mock Data (simulates backend API responses)        |
  +-------------------------------------------------------------------+

Data Flow:
  - TaskContext provides global state to all components
  - Actions (add, edit, delete, toggle) dispatch to a reducer
  - Components consume state via the useTasks() custom hook
  - Mock data simulates what would come from a MongoDB backend

6. MODULE DESCRIPTION
================================================================================

6.1 Layout Module
-----------------
  - TopNavbar: Dark-themed navigation bar with brand logo, page links,
    and a notification dropdown showing urgent deadlines with badge count.
  - Sidebar: Left navigation panel with links, pending task counts,
    overdue badge, and a color-coded source legend (College, Internship,
    Personal). Hidden on mobile for responsive design.
  - Layout: Wrapper component composing Navbar + Sidebar + main content.

6.2 Calendar Dashboard Module
------------------------------
  - CalendarDashboard: Full-featured calendar using react-big-calendar
    with month, week, day, and agenda views. Events are color-coded by
    urgency level. Includes summary stat cards (Pending, Critical,
    Approaching, Safe) and an upcoming deadlines list.
  - TaskQuickView: Modal popup triggered when clicking a calendar event,
    displaying task details with the option to mark complete.

6.3 Task Management Module
---------------------------
  - TaskList: Main task management page with search bar, filters
    (source, priority, status), sorting options (deadline, priority,
    title), and a grid of task cards. Includes add/edit modal trigger.
  - TaskCard: Individual task display card showing title, description,
    deadline, and badges for source, priority, and urgency. Action
    buttons for complete, edit, and delete.
  - TaskForm: Modal form for creating and editing tasks with fields
    for title, description, source, priority, deadline, and link.

6.4 Productivity Analytics Module
----------------------------------
  - AnalyticsDashboard: Statistics overview with four KPI cards (Total,
    Completed, Missed, Completion Rate). Charts include:
      * Weekly Activity (grouped bar chart)
      * Source Distribution (donut pie chart)
      * Monthly Trend (area chart)
      * Priority Distribution (pie chart)

6.5 Microsoft Teams Integration Module
---------------------------------------
  - TeamsIntegration: Two-state component:
      * Disconnected: Shows connect prompt with Microsoft sign-in button
      * Connected: Displays synced meetings (with organizer, time,
        duration, channel, recurring badge) and planner tasks (with
        bucket status and due dates). Includes manual sync button.

6.6 Reminders Module
---------------------
  - ReminderSettings: Configuration panel for browser notifications
    (with permission request) and email alerts. Includes default
    reminder time selector (15min to 1 day). Shows overdue alert
    banner and a list of upcoming reminders sorted by deadline.

6.7 State Management Module
-----------------------------
  - TaskContext: Centralized state using React Context API with
    useReducer. Supports four actions: ADD_TASK, EDIT_TASK,
    DELETE_TASK, TOGGLE_COMPLETE. Provides the useTasks() hook
    for component consumption.

6.8 Utilities Module
---------------------
  - helpers.js: Utility functions including:
      * getUrgencyLevel() - calculates urgency based on time remaining
      * getUrgencyBadge() - returns badge color/label for urgency
      * getSourceBadge() - returns badge color/label for source
      * getPriorityBadge() - returns badge color/label for priority
      * formatDeadline() - formats date for display
      * generateId() - creates unique IDs for new tasks

11. CONCLUSION
================================================================================

The Unified Deadline Management System (DeadlineHub) successfully addresses
the problem of managing scattered deadlines across multiple platforms. The
frontend implementation provides:

  - A visually intuitive calendar dashboard with multiple view options
  - Comprehensive task management with smart urgency classification
  - Productivity analytics for self-assessment and improvement
  - Microsoft Teams integration readiness for organizational workflows
  - Configurable reminder and notification system

The application is built with modern web technologies (React 19, Bootstrap 5,
Vite) following component-based architecture principles. The responsive design
ensures accessibility across desktop and mobile devices. The modular structure
allows easy extension with a backend system for full-stack deployment.

The project demonstrates practical application of:
  - React component composition and state management
  - Context API with useReducer for scalable state handling
  - Third-party library integration (calendars, charts, date utilities)
  - Responsive UI design with Bootstrap grid system
  - Clean code organization and separation of concerns
